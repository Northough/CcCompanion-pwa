# 指令浮窗（无后端方案）

云端的 Claude 通过在**正常聊天回复里嵌入一个记号**给用户下达带倒计时的小任务，
前端解析后弹出浮窗、跳动倒计时；用户点完成/取消后，会以一条普通聊天消息回传，
Claude 借此知道她做得怎么样。整条链路寄生在现有 `/chat` 管道上，**不需要任何新后端**。

## 记号协议

在回复正文的任意位置写：

```
[[task:任务标题:倒计时秒数]]
```

- `倒计时秒数` 可省略：`[[task:读完这一段]]` = 只计时、不倒计时。
- 记号本身不会显示在聊天里（前端会抹掉），所以可以自然地写在句子中间：
  - `好，给你 25 分钟，读完记得回来。[[task:读完这一段:1500]]`
  - `那先专注一会儿吧 [[task:整理笔记]]`

## 回传（用户完成/取消时自动发生）

- 完成：`✓ 完成了『读完这一段』· 用时 24分3秒 · 比预定快 59秒`
- 取消：`✕ 取消了『读完这一段』`

这条会作为普通用户消息进入对话，Claude 会看到并可以自然回应。

## 让伴侣 Claude 学会下达任务

**推荐：用 Skill。** 仓库里已带一个 `.claude/skills/issue-task/SKILL.md`，
它包含记号协议、时长参考、使用时机等完整指引，Claude 会在合适时机自动触发，
协议细节只在真要下任务时才进上下文，日常聊天不占地方。

- 若你的伴侣 Claude 跑在**本仓库**里，开箱即用。
- 若跑在别的目录，把 `issue-task/` 拷到那个项目的 `.claude/skills/`，
  或拷到 `~/.claude/skills/` 做成全局可用。

**或者（更轻）：** 在它读取的 CLAUDE.md / system prompt 里加一句，与其它准则并列：

> 合适时机你可以用 `[[task:标题:秒]]` 给我下达一个带倒计时的小任务，会以浮窗形式显示在我的设备上。倒计时秒数可省略（只计时）。"合适时机"你自己判断，不用每次都问我。我完成或取消后会有一条回执消息回到对话里。

## 前端实现位置

- `client/src/api/commandTransport.ts` — 本地指令存储（localStorage 持久化，刷新可恢复进行中的倒计时）
- `client/src/components/CommandFloatWindow.tsx` / `.css` — 浮窗组件
- `client/src/ChatView.tsx` — 解析 Claude 消息里的记号 → `addCommand`；完成/取消 → 发回执消息
- `client/src/displayText.ts` — 从聊天显示里抹掉 `[[task:...]]` 记号
- 设置页「指令浮窗 · 演示模式」：开启后自动推一条测试指令，用于无 Claude 时调界面（刷新生效）
